# Ticket System - Strapi Backend

The **Strapi Backend** serves as the API and content management layer for the Ticket System. It provides a flexible, secure, and easily manageable infrastructure for handling support ticket data.

## üöÄ Quick Start

### 1. Prerequisites
- Node.js >= 20.x
- npm or yarn

### 2. Installation
```bash
npm install
```

### 3. Development
Start the server in development mode (with auto-reload):
```bash
npm run develop
```
- **Admin Panel**: [http://localhost:1337/admin](http://localhost:1337/admin)
- **API**: [http://localhost:1337/api](http://localhost:1337/api)

### 4. Build
Build the admin panel for production:
```bash
npm run build
```

## üõ†Ô∏è Architecture

This project uses **Strapi v5** with a **PostgreSQL** database (in production) and **SQLite** (in development).

- **Controllers**: Custom logic in `src/api/[content-type]/controllers/`
- **Services**: Reusable business logic in `src/api/[content-type]/services/`
- **Routes**: API endpoints defined in `src/api/[content-type]/routes/`
- **Policies**: Middlewares for security and validation.

For a deep dive, see the **[BMAD Architecture Docs](BMAD/_bmad-output/architecture.md)**.

## üì¶ Deployment (Render)

This project is configured for deployment on **Render**.

**Key Environment Variables:**
- `NODE_ENV`: `production`
- `DATABASE_URL`: Connection string.
- `APP_KEYS`, `API_TOKEN_SALT`, `ADMIN_JWT_SECRET`: Security secrets.

See the **[Development Guide](BMAD/_bmad-output/development-guide.md)** for detailed deployment instructions.
